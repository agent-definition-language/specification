# Proposals

This directory is for **proposals** that affect the Agent Definition Language (ADL) specification or the standardization process.

## What Belongs Here

- **Specification change proposals** — New or changed sections of the spec (e.g., expanding the Identity, Capabilities, Tools, or Policies objects).
- **Standardization submissions** — Draft text or packages for submitting ADL to a standards body (e.g., Linux Foundation, IETF, ISO).
- **Process or governance changes** — Changes to CONTRIBUTING, GOVERNANCE, or release process.

## Format

- One document (Markdown preferred) per proposal.
- Name files descriptively: `YYYY-MM-DD-short-title.md` or `NNN-short-title.md` if you use issue numbers.
- Include:
  - **Summary** — One paragraph.
  - **Motivation** — Why this change or submission.
  - **Details** — Technical or process details.
  - **Alternatives** — Considered alternatives.
  - **References** — Links to issues, discussions, or external docs.

## Workflow

1. Open an issue (or discussion) to gather feedback.
2. Add a proposal document in this directory.
3. Open a pull request that links the proposal to the issue.
4. After review, the change may be merged into the spec or standardization materials (see [GOVERNANCE.md](../GOVERNANCE.md)).

See [CONTRIBUTING.md](../CONTRIBUTING.md) for general contribution guidelines.
